Definition and Characterization: Deadlock is a permanent state where processes are waiting indefinitely for resources held by each other. Know examples (e.g., dining philosophers).
Necessary Conditions (Coffman's Conditions): All four must hold – Mutual Exclusion (resources can't be shared), Hold and Wait (process holds resources while waiting for others), No Preemption (resources can't be forcibly taken), Circular Wait (cyclic dependency).
Resource Allocation Graph (RAG): Directed graph with processes (circles), resources (squares), request edges (process to resource), assignment edges (resource to process). Deadlock if cycle exists and no way to reduce (for single-instance resources).
Deadlock Prevention: Eliminate one condition – e.g., allow sharing (no mutual exclusion), request all resources at start (no hold and wait), allow preemption, or order resources hierarchically (no circular wait).
Deadlock Avoidance: Runtime check to ensure safe state. Know Banker's Algorithm: Safe state (exists a sequence where all processes can finish), Need Matrix (Max - Allocation), Safety Algorithm (simulate allocation), Resource-Request Algorithm. Practice numericals with small data.
Deadlock Detection: Periodic check for cycles in RAG or using a variant of Banker's. For multiple instances, use wait-for graph.
Deadlock Recovery: Once detected, recover via process termination (victim selection: least cost, youngest, etc.), resource preemption (rollback), or ignore (ostrich algorithm).
Related Concepts: Starvation (indefinite postponement, e.g., due to priority), Livelock (processes change states but no progress), Thrashing (related to paging but sometimes confused).
Handling in Real OS: Examples like Windows/Linux deadlock detection tools, avoidance in practice (rare due to overhead).
Practice Tips: Solve 50+ MCQs from GeeksforGeeks, GATE papers (1990-2020), and CDAC-specific samples. Focus on why options are wrong. For numericals, practice 5-10 Banker's problems.
